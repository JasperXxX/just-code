03Sound & status — the part that "speaks" to you

You mentioned "the speaking" — this is the audible feedback system.

Random sound on stop & notification

Every time I finish or a notification fires, play-random-sound.sh runs and plays a random sound from ~/Sounds/claude/.

sounds=($HOME/Sounds/claude/*)
sound="${sounds[$((RANDOM % ${#sounds[@]}))]}"
afplay -v "$volume" "$sound" &

Volume auto-adjusts: quieter (0.6) when other audio is playing, full (1.0) otherwise.

Status line at the bottom

The line at the bottom shows:

user@host:/current/path   [Opus 4.7 (1M context) | ctx: 73% left]

Script: ~/.claude/statusline-command.sh — shows user, hostname, CWD, model, remaining context.

Caffeinate loop

While I'm working, your Mac stays awake. On submit, caffeinate -dimsu starts; on stop, it gets killed. Tracked via session PID file in /tmp/claude-caffeinate-$SID.pid.

Push notifications

agentPushNotifEnabled: true — you get pushes from the mobile companion.

09Memory

Two layers — short facts per project, plus cross-project working-style files.

Auto-memory · per project

Path: ~/.claude/projects/<encoded-cwd>/memory/ — the CWD path with / replaced by -

Content: short, structured facts with frontmatter. One MEMORY.md as index, plus one .md per entry.

Types:

  • user — who you are, role, preferences
  • feedback — corrections + validations, with Why + How to apply
  • project — facts about ongoing work, with absolute dates
  • reference — pointers to Linear, Slack, dashboards, repos
Working-style files · global

Path: ~/.claude/working-style/*.md — indexed in the global CLAUDE.md, loaded on demand

Content: cross-project patterns — how you communicate, which tools/skills you use, feedback log, recurring workflows.

Example files:

  • prompt-style.md — language mix, tone, signals
  • tool-usage.md — skill/tool usage, friction
  • feedback-log.md — dated stream of "worked / didn't"
  • patterns.md — recurring workflows

Persist a note with # at the start of a line → lands in CLAUDE.md or memory. /extra is for the current conversation only. Sanitized examples live under claude/memory/ in the repo.

14Shell — Zsh

macOS default Zsh, no oh-my-zsh. .zprofile sets up Brew + Resolve scripting; .zshrc has conda, bun, aliases.

Aliases

PATH order

$HOME/.local/bin → $BUN_INSTALL/bin → conda (miniforge) → Brew (via .zprofile) → system.

DaVinci Resolve scripting (.zprofile)

export RESOLVE_SCRIPT_API="/Library/Application Support/Blackmagic Design/DaVinci Resolve/Developer/Scripting"
export RESOLVE_SCRIPT_LIB="/Applications/DaVinci Resolve/DaVinci Resolve.app/Contents/Libraries/Fusion/fusionscript.so"
export PYTHONPATH="$PYTHONPATH:$RESOLVE_SCRIPT_API/Modules"

conda-init from miniforge is loaded on shell start in .zshrc, otherwise stays out of the way.
